문제
무한히 큰 배열에 다음과 같이 분수들이 적혀있다.

1/1	1/2	1/3	1/4	1/5	…
2/1	2/2	2/3	2/4	…	…
3/1	3/2	3/3	…	…	…
4/1	4/2	…	…	…	…
5/1	…	…	…	…	…
…	…	…	…	…	…
이와 같이 나열된 분수들을 1/1 → 1/2 → 2/1 → 3/1 → 2/2 → … 과 같은 지그재그 순서로 차례대로 1번, 2번, 3번, 4번, 5번, … 분수라고 하자.

X가 주어졌을 때, X번째 분수를 구하는 프로그램을 작성하시오.

입력
첫째 줄에 X(1 ≤ X ≤ 10,000,000)가 주어진다.

출력
첫째 줄에 분수를 출력한다.

## 💻 알고리즘 코드

```javascript
// 백준 스타일 입력 — require("fs").readFileSync 지원
const input = require("fs").readFileSync("/dev/stdin", "utf8").trim();
const X = Number( input.split("\n")[0])

function solution(X){
  let line = 1; // 현재 몇 번째 대각선 라인인지
  let sum = 0;  // 현재 라인까지의 누적 분수 개수

  // 1. X가 속한 라인이 어디인지 찾기
  while (sum + line < X) {
    sum += line;
    line++;
  }

  // 2. 해당 라인 내에서 몇 번째 칸인지 계산 (1부터 시작)
  const targetIndex = X - sum;

  // 3. 짝수 라인 vs 홀수 라인 방향에 따라 결과 계산
  if (line % 2 === 0) {
    // 짝수 라인: 위에서 아래로 (1/line -> line/1)
    // 분자는 증가, 분모는 감소
    return `${targetIndex}/${line - targetIndex + 1}`;
  } else {
    // 홀수 라인: 아래에서 위로 (line/1 -> 1/line)
    // 분자는 감소, 분모는 증가
    return `${line - targetIndex + 1}/${targetIndex}`;
  }
}

console.log(solution(X))
```

## 💡 핵심 포인트

규칙을 잘 찾보자.

규칙 1: $n$번째 라인에는 $n$개의 분수가 있습니다.
규칙 2: 라인 번호($n$)가 홀수면 위로 올라가는 방향($n/1 \to 1/n$), 짝수면 아래로 내려가는 방향($1/n \to n/1$)입니다.
